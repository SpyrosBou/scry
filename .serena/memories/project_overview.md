Scry — automated Playwright-based auditing platform for WordPress sites covering functionality, responsiveness, accessibility, and visual regressions. Node.js 18+, npm, Playwright, Axe, and Sass are the primary tooling. Key folders: `tests/` for Playwright specs plus baseline snapshots, `utils/` for shared helpers (test runner, fixtures, reporting utilities, schema helpers), `sites/` for per-environment JSON configs and manifests, `reports/` and `test-results/` for generated artifacts, and scripts such as `run-tests.js` plus helpers under `scripts/`. Development focuses on reliability of the harness/reporter rather than individual site pass rates; documentation (README, SPEC, AGENTS) must stay updated and include the latest commit hash note.
