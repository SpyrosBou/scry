Four dimensions of quality

Each audit captures a different axis of site health, so nothing slips through.

Functionality

Validates links, interactive elements, and infrastructure health. Catches broken navigation, dead-end forms, and server errors before they reach production.

error 3 broken links found on /products

Responsiveness

Tests layout integrity across mobile, tablet, and desktop viewports. Flags overflow, clipping, and layout shifts that break the experience on real devices.

warning Horizontal overflow at 375px viewport

Accessibility

Runs axe-core audits against WCAG 2.1 criteria. Surfaces violations by impact, maps them to specific DOM nodes, and tracks compliance over time.

pass All 47 WCAG rules passing

Visual Regression

Compares screenshots against baselines pixel by pixel. Detects unintended visual changes from theme updates, plugin conflicts, or CSS drift.

2.4% diff Header layout shifted on /about

Three steps to clarity

From URL to interactive report in minutes.

1

Configure

Point Scry at any WordPress site. Automatic sitemap discovery finds every page — or specify exactly which pages to test.

$ npm run discover -- https://your-site.com
2

Run

Choose your suites and let Playwright do the work. Parallel workers distribute tests across browsers and viewports.

$ node run-tests.js --site your-site --all-suites
3

Review

Open rich, interactive HTML reports. Drill into findings by suite, page, and severity. Track progress across runs.
